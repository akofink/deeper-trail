Discovery date: 2026-03-09

Original note:
It'd be great if the levels were not so flat. There's a good game feel now, and we could lean into the platformer style a bit more.

The obstacles are too plain / are placeholders. Enhance random biome generation with custom obstacles or enemies to avoid. Mix up the types of movement patterns possible for given enemy classes, and add more variety to the types of obstacles that can be generated.

The map is really hard to figure out. The point cloud of nodes results in a path back and forth that is linear but still confusing because the path lines are all going through the middle of the map. We should instead generate subgroups of local points that form sort of districts and are then connected with some inter-district paths. The number of connections should be an element of random generation, resulting in more or less linear maps. The districts should be navigated to for unique rewards or upgrades that aid or are required to pass obstacles to get to the signal source location to ultimately win the game. The randomness is a good start, but it could really be made better to be made more progressive, though it doesn't need to necessarily be linear.

Popup messages are easy to miss. Some sort of local animation (i.e. jump!) could be good. Generally, the prompts are difficult to read, and there is a lot of text players don't read. The anomaly boost + hitting enter beacons can be difficult to understand and figure out but not impossible.

Resolution note (2026-03-09): Triaged this playtest dump into separate actionable issue reports for terrain verticality, biome obstacle variety, map graph readability, and prompt/beacon feedback clarity.
